Linux 变慢优化：
    命令        功能
    top         查看所有进程
    uptime      查看系统负载
    vmstat -n 2 3 查看CPU每2秒采样一次，采样3个 查看进程数阻塞占用空间等信息
    mpstat -P ALL 2 查看所有cpu进程数
    pidstat -u 1 -p 进程编号 查看每一个进程使用cpu用量分析
    free -m        查看内存大小 单位m
    df -h       查看磁盘剩余空间
    iostat -xdk 2 3 磁盘IO性能评估
    ifstat           查看网络io

    常用命令：

        1，lsof -i:端口号

        2，netstat -tunlp|grep 端口号

        这两个命令都可以查看端口被什么进程占用


        2、查看服务器所有端口

        netstat -ntlp
    


    java cpu使用过高：
        1.jps -l      获取进程id（pid）
        2.top -Hp pid
          或
          ps -mp pid -0 THREAD,tid,time|sort -k 2 -r
        3.jstack pid >日志名字
        4.把刚才第二步排查出来的 使用率最高的线程 转为16进制 前面加上 0x然后去日志里搜索


    分布式集群实现数据拷贝 scp 

      scp    -r        $pdir/$fname             $user@$host:$pdir/$fname
      命令   递归     要拷贝的文件路径/名称   目的地用户@主机:目的地路径/名称

      e.g. 
        scp -r liuweiyi@192.168.10.102:/opt/module/* liuweiyi@hadoop104:/opt/module


     远程同步工具 rsync
      rsync主要用于备份和镜像。
      rsync 和 scp 区别：
        1.rsync比scp快
        2.rsync差异化更新，scp更新全部的

        rsync    -av       $pdir/$fname             $user@$host:$pdir/$fname
        命令   选项参数   要拷贝的文件路径/名称   目的地用户@主机:目的地路径/名称

        e.g.
          rsync -av liuweiyi@192.168.10.102:/opt/module/hadoop-3.1.3/ atguigu@hadoop103:/opt/module/hadoop-3.1.3/
